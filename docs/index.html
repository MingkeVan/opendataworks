核心功能

为企业数据平台提供全方位的管理和监控解决方案

📊

数据资产管理

统一管理数据表元信息、字段级别管理、数据域和业务域分类，支持 ODS/DWD/DIM/DWS/ADS 完整分层

🔄

任务调度编排

可视化配置批处理和流处理任务，深度集成 DolphinScheduler，支持 SQL 和 Shell 任务类型

🔗

数据血缘追踪

基于任务输入输出自动生成血缘关系图，ECharts 力导向图可视化展示，支持链路追踪

📈

执行监控

实时监控任务执行状态，查看执行历史和日志，统计任务成功率和耗时，失败任务告警

🎯

智能调度

自动查询 DolphinScheduler 项目信息，无需硬编码，支持任务依赖配置，工作流上线/下线管理

快速开始

根据场景选择合适的部署方式，一页即可完成准备工作

⏱️ 约 5 分钟 🧰 适用场景：首次体验 / 快速演示 
# 克隆项目并进入目录
git clone https://github.com/MingkeVan/opendataworks.git
cd opendataworks

#!/bin/bash
# 初始化数据库（自定义 root 与应用密码）
./init-database.sh -r <mysql-root> -p <app-password>

# 一键启动前后端与编排服务
./quick-deploy.sh --with-doris=false

# 浏览器访问
# 前端:           http://localhost:3000
# 后端 API:       http://localhost:8080
# DolphinScheduler: http://localhost:12345/dolphinscheduler
⏱️ 约 8 分钟 🧰 适用场景：CI/CD 集成 / 云主机部署 
# 克隆仓库并进入目录
git clone https://github.com/MingkeVan/opendataworks.git
cd opendataworks

cp sample.env .env # 根据环境修改变量

# 启动服务（前后端、MySQL、DolphinScheduler）
docker compose -f docker-compose.yml up -d

# 查看运行状态
docker compose ps

# 停止服务
docker compose down

# 可选：集成 Doris
# docker compose -f docker-compose-doris.yml up -d
⏱️ 约 12 分钟 🧰 适用场景：二次开发 / 团队协作 
# 准备前端环境
cd frontend
npm install
npm run dev -- --host 0.0.0.0 --port 3000

# 新开终端：启动后端
cd backend
mvn clean install -DskipTests
mvn spring-boot:run

# 启动 DolphinScheduler 服务
cd ../dolphinscheduler-service
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 9000

准备阶段 建议：8 核 / 16 GB

确认 MySQL、Docker 环境或 JDK 8+ 等依赖已安装，可选创建独立的操作系统用户。

部署阶段

按照上方选定的方式完成部署，等待服务端口全部就绪后执行健康检查脚本 scripts/check-services.sh。

体验阶段

登录 DolphinScheduler 创建首个流程，使用前端血缘图谱确认任务运行情况，并在首页仪表盘查看实时指标。
